# Documentation Technique

## Objectif

Ce dossier regroupe les livrables techniques du projet.

## Arborescence

- `architecture/` : schemas, cadrage d'infrastructure, flux, adressage
- `procedures/` : installation, exploitation, verification, maintenance
- `captures/` : captures d'ecran et exports utiles a la soutenance
- `templates/` : modeles Markdown reutilisables

## Regles d'usage

- un document par sujet fonctionnel
- un titre explicite
- une date de mise a jour
- une section "Hypotheses" si des arbitrages sont poses
- une section "Validation" quand une procedure est testee
